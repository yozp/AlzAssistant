# 阿尔茨海默症智能问答系统 - 模型微调数据集

## 数据集说明

本目录包含用于Llamafactory模型微调的高质量数据集，涵盖阿尔茨海默症智能问答系统的各种对话场景。

## 数据集文件

### 1. symptom_assessment.json
**症状评估数据集**
- **数量**：10条
- **内容**：包含各种症状描述场景，AI需要根据症状进行专业分析和评估
- **场景**：
  - 早期记忆问题
  - 执行功能下降
  - 精神症状（妄想、幻觉）
  - 空间定向障碍
  - 语言障碍
  - 情绪和行为改变
  - 判断力下降
  - 记忆混乱

### 2. knowledge_qa.json
**知识问答数据集**
- **数量**：10条
- **内容**：基于知识库的常见问题解答
- **场景**：
  - 疾病基础问题（什么是AD、与痴呆的区别等）
  - 遗传相关问题
  - 治疗相关问题
  - 预防相关问题
  - 照护相关问题

### 3. professional_advice.json
**专业建议数据集**
- **数量**：7条
- **内容**：提供专业的就医建议、生活方式建议、照护建议和预防建议
- **场景**：
  - 就医建议（科室选择、准备事项）
  - 生活方式建议（认知活动、运动、饮食等）
  - 照护建议（中期阶段照护）
  - 预防建议（降低风险）
  - 药物治疗建议
  - 情绪问题管理
  - 早期筛查和监测

### 4. multi_turn_conversation.json
**多轮对话数据集**
- **数量**：10条
- **内容**：模拟真实的多轮对话场景
- **场景**：
  - 症状咨询的多轮对话
  - 治疗建议的多轮对话
  - 照护问题的多轮对话
  - 预防咨询的多轮对话
  - 药物疗效讨论
  - 重复询问的处理
  - 基因检测咨询
  - 照护者支持
  - 睡眠问题管理
  - 疾病发展过程咨询

### 5. comprehensive_qa.json
**综合问答数据集**
- **数量**：7条
- **内容**：涵盖更多样化的问答场景
- **场景**：
  - 疾病预后相关问题
  - 日常生活相关问题（开车、工作、旅行）
  - 照护机构选择
  - 费用相关问题
  - 研究进展
  - 旅行相关问题

## 数据格式

所有数据集使用**Alpaca格式**，符合Llamafactory的标准格式：

```json
{
  "instruction": "系统提示词，定义AI的角色和任务",
  "input": "用户输入的问题或场景描述",
  "output": "AI的专业回复，使用Markdown格式"
}
```

## 数据特点

1. **专业性**：所有回答基于专业的医学知识库，确保准确性
2. **结构化**：回答使用Markdown格式，结构清晰
3. **完整性**：包含症状分析、评估、建议等完整内容
4. **安全性**：所有回答都包含免责声明，强调不能替代医疗诊断
5. **多样性**：涵盖症状评估、知识咨询、专业建议、多轮对话等多种场景

## 使用说明

### 1. 数据合并

如果需要合并所有数据集进行训练，可以使用以下Python脚本：

```python
import json
import glob

# 读取所有JSON文件
files = glob.glob("*.json")
all_data = []

for file in files:
    if file != "README.md":
        with open(file, 'r', encoding='utf-8') as f:
            data = json.load(f)
            all_data.extend(data)

# 保存合并后的数据
with open("merged_dataset.json", 'w', encoding='utf-8') as f:
    json.dump(all_data, f, ensure_ascii=False, indent=2)

print(f"合并完成，共 {len(all_data)} 条数据")
```

### 2. Llamafactory配置

在Llamafactory中使用时，需要：

1. **数据格式**：确保使用Alpaca格式
2. **数据路径**：指定数据集文件路径
3. **训练参数**：根据模型大小和计算资源调整

示例配置：

```yaml
dataset: alz_assistant_dataset
template: default
cutoff_len: 2048
overwrite_cache: true
```
